## Entscheidungen

### Entscheidung 1: SECURITY-Reporting via GitHub Private Advisories
Statt eines Phantom-`security@lokyy.local` Placeholder hat Paige direkt die moderne GitHub-Flow als einzigen Reporting-Channel eingebaut. Vermeidet broken mailto + matched 2026 Best Practice. Oliver kann nachträglich ein Mailbox dazustellen (2-Liner) falls gewünscht.

### Entscheidung 2: SVG-Renderer als persistentes Tool
`scripts/generate-lokyy-icons.ts` wandert in `scripts/`, nicht in `_throwaway/`. Idempotent — wenn das Logo später iteriert (Phase E.2 oder Etappe 2), regeneriert das Tool das ganze Set mit einem Befehl.

### Entscheidung 3: Hermesworld-Assets löschen, Routen behalten
Brief-Recommendation "hide" → Sidebar zeigt Marketing-Routen nicht. Wenn jemand direct URL nutzt, gibt's 404 für die gelöschten Bilder. Trade-off: 24 MB Repo-Cleanup > graceful failure auf hidden routes. Dokumentiert als Q-004 in `docs/questions.md`.

### Entscheidung 4: In-Component-UI-Strings als Etappe-2-Carry-Over
Phase E hat **Asset + README + Doku-Strings** behandelt. Inside-Komponenten-JSX-Strings ("HERMES WORKSPACE" im Chat-Empty-State, Login-Modal, Onboarding, Mobile-Tab-Bar-Labels usw.) waren nicht im Brief-Scope. Dokumentiert als Q-003. **Empfehlung an Oliver:** eigenes Etappe-2-Issue für "Komplett-Sweep aller sichtbaren UI-Strings".
