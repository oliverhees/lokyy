## Decisions + Rationale

### D1 — lokyy-brain bleibt eigener Service (NICHT nativ integrieren)
**Rationale**: Sunk-cost recovery + Karpathy-Doktrin "markdown is the state" macht Service-Boundary natürlich + lokyy-brain bleibt von anderen Apps nutzbar (PWA, externe, MCP) + Epic-5 (MCP) wird natürlicher Agent-Zugang. Advisor-Cross-Check bestätigt.

### D2 — Hermes Agent (nousresearch/hermes-agent) als Core
**Rationale**: WebFetch verifiziert — Hermes ist ein vollwertiges Agent-Framework mit eingebautem Cron-Heartbeat, autonomous-skill-creation, Multi-Agent-Subagent-Spawning, 40+ Tools, model-agnostic, Docker-deploy, MIT. Lokyy wird Gateway-Layer um Hermes herum, NICHT eigene Agent-Engine.

### D3 — Three-Layer-Heartbeat-Architecture
**Rationale**: Korrektur nach User-Feedback. Hermes-Cron plant Tasks, aber ohne Lokyy-Supervisor läuft autonom nichts wirklich autark (Crash, Hang, Server-Sleep, Missed-Cron = Task-Rot). Layer-1 Specialists (wake/sleep, hybrid pull+push), Layer-2 Hermes (plant+delegiert), Layer-3 Lokyy-Supervisor (überwacht Liveness, auto-restart). Klare Owner-Matrix: Supervisor schedulet NIE selbst.

### D4 — Docker-first auf Server, Reverse-Proxy = Traefik
**Rationale**: Alle Komponenten in Containern. Traefik wegen Docker-Label-Auto-Discovery (aligned mit Phase-8 Docker-MCP-spawned-services), built-in Dashboard, auto-TLS via Let's Encrypt.

### D5 — TelosAdapter Interface, read-only Volume MVP
**Rationale**: PAI Telos via read-only Docker-Volume in lokyy-os-be (single-user, same-host MVP). Adapter-Interface erlaubt späteren Swap zu sync-copy / git-pull ohne Agent-Touch.

### D6 — OpenClaw als peer-Agent-Bridge (nicht Coding-Tool)
**Rationale**: OpenClaw ist laut User + Recherche kein Coding-Agent sondern peer-Agent-System (Peter Steinberger, MIT). Phase-6-Bridge wird Drei-Fach: Lokyy-native (Hermes), Claude Code (subprocess), OpenClaw (HTTP/Messaging).

### D7 — Install-Wizard als Phase-9
**Rationale**: System muss auf Fresh-Server per Wizard installierbar sein. `lokyy-installer` CLI: install / up / down / purge / status. Idempotent. Parallel zu Phase-1+ implementierbar.
