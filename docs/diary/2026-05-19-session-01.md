## Lessons Learned

- **Recon vor Implementation**: 3 initiale P0-Issues (#115/#116/#117) hatten falsches Scope ("just swap stub→proxy"). Tatsache: hermes-dashboard hat keine JSON-API dafür. Recon-Update nach 1 PR sparte sich Stunden Fehlweg.
- **Agent-Briefings müssen Verify-Step explizit machen**: Tasks-Engineer schrieb Code aber commit + Playwright übersprungen. Frontend-Developer-Agent (Settings) hat's autonom durchgezogen. Beide kriegten dasselbe Brief-Template — Unterschied liegt im Agent-Profile.
- **docker-exec ist mächtiger als gedacht**: Mit `EXEC=1` am proxy + container-name-scoping in code wurde aus "stub" "live CLI integration" — wiederverwendbar für tools/webhooks/memory/curator/doctor.
- **Curated lists > 0-Empty-State**: Wenn die echte Liste hardcoded in Hermes-Source ist (Channels/Integrations), Lokyy curaten + `configured` dynamisch aus config.yaml. User sieht was POSSIBLE ist statt was AKTIVIERT ist.
